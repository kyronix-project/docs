# Contributing

This section describes how to contribute to the Kyronix kernel project. Kyronix welcomes contributions from both human developers and AI-assisted development.

## Overview

Kyronix is an ISC-licensed, x86-64 hybrid Linux-compatible operating system. Contributions are accepted in the form of patches, bug reports, and documentation improvements.

## Guidelines

1. [Contribution Overview](overview.md) -- general workflow and process
2. [Commit Messages](commit-messages.md) -- format and conventions
3. [Coding Style](coding-style.md) -- code formatting rules
4. [AI Policy](ai-policy.md) -- rules for AI-generated contributions

> **Important:** All contributions must follow the coding style and commit message conventions. Patches that do not conform may be rejected.
